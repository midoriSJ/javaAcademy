@ 예외 처리

에러 : 하드웨어 고장으로 인해 응용 프로그램이 실행 오류 발생
- 외부적인 상황으로 코드랑은 상관없이 고장이 생기는 환경이다.
(개발자도 대체할 수 없다.)
- 인터럽트 / 입출력 / 메모리 고장

예외 : 개발자의 잘못된 코딩으로 인해서 발생하는 에러를 말한다.
예외가 발생하면 프로그램은 곧바로 종료된다.

- 일반 예외 : 컴파일러가 무조건 예외 처리 검사하는 방식
1. 자바 컴파일러가 명시적으로 블록을 꼭 사용해서 요구하는 예외 상황
(try - catch - finally) : 선택 X, 무조건 처리 O

- 실행 예외 : 컴파일러가 예외 처리 코드 여부를 검사하지 않는 방식
(try - catch - finally) : 선택 O, 무조건 처리 X
- NullpointerException : 객체 인스턴스를 생성했는데 메모리가 없는 경우
- NumberFormatException : 잘못된 형 변환 (정수 / 실수 단위로 달라지는 부분)
- ArrayIndexOutOfBoundsException : 배열의 범위가 없는데 배열의 주소를 초과해서 발생하는 경우

* 예외 처리 : 클래스로 구성되어 있다.
일반 예외 > Exception > throwable 상속 받아 만들어진다.
실행 예외 > RuntimeException > throwable 상속 받아 만들어진다.
